# Python SDK Status

Python bundles run on the same runtime capability layer as JavaScript and TypeScript.

Current status:

- Python plugin entrypoints are supported
- Python receives the same service-oriented runtime context shape
- the CLI can scaffold and bundle Python projects

Current limitation:

- there is still no dedicated Python package equivalent to `@lapis-lazuli/sdk`

Practical meaning:

- TypeScript is the primary API design target
- Python support is runtime-capable but still early from an authoring-tooling perspective
